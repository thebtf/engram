## Overview

Fix 8 findings from the post-implementation investigate report. 3 P1 (must fix) + 5 P2 (should fix).

## Functional Requirements

### FR-1: Summary Dedup (P1, F-0-11)
Server-side summarizer (maintenance Task 19) must check for existing summaries before generating.
The `summarizeUnsummarizedSessions` query already has `NOT EXISTS (SELECT 1 FROM session_summaries
WHERE sdk_session_id = s.claude_session_id)` — verify this works correctly.

### FR-2: Summary Verification (P1, F-0-10)
Trigger maintenance on deployed server and confirm summaries appear in dashboard.
If server not yet deployed with v2.2.0, this is blocked on Docker CI.

### FR-3: OpenClaw before_tool_call Return Type (P1, F-0-2)
The `handleBeforeToolCall` returns `{ appendSystemContext }` but OpenClaw `HookResult` type
doesn't include this. Either update the type definition or use a supported return mechanism.

### FR-4: Store Empty Content Validation (P2, F-0-1)
`store(action="create")` should reject requests without content. Add validation in
`handleStoreMemory` or `handleStoreConsolidated`.

### FR-5: Summary userPrompt Threshold (P2, F-0-3)
Lower the 50-char threshold for userPrompt fallback in ProcessSummary. Short prompts
like "fix this" (8 chars) should still produce a summary.

### FR-6: Concept Backfill Completion (P2, F-0-9)
Add keywords for the 5 missing concepts: why-it-exists, what-changed, anti-pattern,
gotcha, trade-off. Either new migration or update existing.

### FR-7: Behavioral Rules Effectiveness Metric (P2, F-0-8)
The effectiveness metric for always-inject rules measures citation, not compliance.
Document this limitation or add a compliance-based metric.

### FR-8: Dashboard Visual Verification (P2, F-1-13)
Verify dashboard visually via Playwright or screenshot comparison.
